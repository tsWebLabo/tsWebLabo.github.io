Androidアプリ制作に挑戦！！

YouTubeライブ配信でつくったスマホアプリを紹介します。ソースコードやスクリーンショットを掲載していきます。

ここではAndroid Studioを使ってアプリを作成する時に便利で基本的な機能の実装について紹介します。
ひとつひとつは簡単なものですが組み合わせることで様々な可能性を見いだせます。

Basic-Tips

UbuntuでのAndroid Studioインストールと設定

UbuntuでAndroidStudioを使う際、アプリのテストでエミュレータを使います。
そのためソフトウェアのインストールの他にいくつか設定が必要になります。
エミュレータを使わず実機テストのみでAndroidStudioを使う場合は必要ありません。

  1. BIOSの設定で仮想マシンの使用を許可する

    2. PC起動時に[F2]や[Delete]等のキーを押すことでBIOS設定画面に入ることができます。

    注意！！：BIOSの設定を不用意に変えてしまうのは非常に危険です。
    僕の動画やWebサイト以外の情報も取り入れて自己責任で行ってください。

    • Advanced -> CPU Configuration -> Execute Disable Bit -> "Enabled"
    • Advanced -> CPU Configuration -> Intel(R) VirtualizationTechnology -> "Enabled"
  2. 仮想マシンKVMをインストールして使えるようにする
    • UbuntuにKVMをインストール
      •                       
                        sudo apt install qemu-kvm libvirt-bin ubuntu-vm-builder bridge-utils
    • 一般ユーザーをKVMグループに所属させる
      • KVMの所有者を確認する
        •                           
                            ls -al /dev/kvm
      • kvmグループに属しているユーザーを確認する
        •                           
                            grep kvm /etc/group
      • kvmグループに自分の使っているユーザーを追加する
        •                           
                            sudo adduser $USER kvm
Android Studioの起動からHello World!!へ
Android Studioの起動

AndroidStudioのアイコンをダブルクリックするとAndroid Studioが起動し、左のような画面が出てきます。
「Create New Project」をクリックして新規アプリをつくってみましょう！！

新規アプリのアクティビティ選択

新規アプリのテンプレートとなるアクティビティを選択できます。
ここでは「Hello World」の表示を目指すので、シンプルで基本的な「Enpty Activity」を選択しました。

新規アプリ作成の詳細設定

起動画面の「Create New Project」をクリックすると左の画面が表示されます。
画像右側の入力フォームにそれぞれ必要事項を入力します。

  • Name：新規アプリの名前を入力
  • Package name：新規アプリのパッケージ名。自分のメールアドレスとアプリ名を組み合わせたもの。自動的に入力される
  • Save location：データの保存場所。インストール作業中に保存場所を聞かれるのでそのままでOK
  • Language：JavaとKotlinのどちらかを選択可能。
  • minimum SDK：AndroidOSのバージョン指定
    • 新しいバージョンで指定すると対象となるユーザーは比較的限定されてしまうが、新機能を取り込んだアプリを作成できる。
    • 古いバージョンで指定すると新機能を取り込んだアプリを作成できないが、より多くのユーザーを対象としたアプリを作ることができる
全ての入力が確認できたら、Finishを押しましょう。

メイン画面が表示される

左の画像と同じ表示がされれば、新規アプリのテンプレートができています。
画面右上に"Code","Sprit","Design"というタブがあるので"Sprit"を選択すると左の画像のような配置になります。

問題なく表示されたら、画面上部にあるツールバーの「実行ボタン（緑色の再生ボタンぽいもの）」をクリックしましょう。 エミュレータが起動してスマートフォンでの見た目や機能が再現されます。

Androidエミュレータの起動

Android SDKやAndroid AVDの設定等が済んでいれば、別ウィンドウが開いて、スマートフォンの画像が表示されたと思います。
左の図はその一例です。

  • SDK：Android OSのバージョン管理ツール
  • AVD：エミュレータとして起動する端末情報の管理ツール
SDKとAVDの整合性が取れている必要があります。詳しくは別の項目で解説する予定です。

エラーなく表示されればそこに「Hello World」と表示されているはずです。
さぁ、ここからがアプリ作成のスタートラインです。

アクティビティ編集画面の切り替えタブ
アクティビティ編集画面の切り替えタブ

赤い楕円で囲った部分に注目してください。「Code」「Sprit」「Design」の3つのタブがあります。
この3つのタブは、Android Studioでアプリを作る際に非常に便利な機能です。
ひとつずつ見ていきましょう。

Codeタブ

Codeタブを選択すると、前の画像のような2分割の画面ではなくコードのみのレイアウトになります。
このレイアウトでは、1行が長く折り返してしまうような部分（階層の深いパスやURLなど）を見やすくしたい時に便利です。

Spritタブ

Spritタブを選択すると、コードとデザインの両方が左右に2分割で表示されるためXMLを編集しながら実際の見た目を確認できます。
個人的に3つのレイアウトの中で最も使いやすいレイアウトだと思っていて、僕は大抵の作業をこのレイアウトで行っています。

Designタブ

Designタブを選択すると、画面中央にアプリのデザインが表示され様々なオブジェクトをマウス操作で配置・調整できます。
XMLでの編集にまだ慣れていない方や、直感的にデザインの調整等を行いたい方にオススメのレイアウトです。

画面遷移の実装

AndroidアプリはPCと比較して画面の小さい端末で操作するため画面遷移の機能を使うことでより豊富な機能を実装させることができます。
実際のソースコードから見てみましょう。

activity_main.xml

                  
                    <?xml version="1.0" encoding="utf-8"?>
                    <androidx.constraintlayout.widget.ConstraintLayout xmlns:android="http://schemas.android.com/apk/res/android"
                        xmlns:app="http://schemas.android.com/apk/res-auto"
                        xmlns:tools="http://schemas.android.com/tools"
                        android:layout_width="match_parent"
                        android:layout_height="match_parent"
                        tools:context=".MainActivity">

                        <TextView
                            android:layout_width="wrap_content"
                            android:layout_height="wrap_content"
                            android:text="画面１"
                            android:textSize="40dp"
                            app:layout_constraintBottom_toBottomOf="parent"
                            app:layout_constraintLeft_toLeftOf="parent"
                            app:layout_constraintRight_toRightOf="parent"
                            app:layout_constraintTop_toTopOf="parent" />

                        <Button
                            android:layout_width="match_parent"
                            android:layout_height="wrap_content"
                            android:text="画面２へ"
                            android:textSize="40dp"
                            android:onClick="onButtonIntent1" // この指定でこのボタンが押された時の処理をjavaファイルに書くことができる。
                            app:layout_constraintBottom_toBottomOf="parent" />

                    </androidx.constraintlayout.widget.ConstraintLayout>
ButtonにonClickを指定する

android:onClick="onButtonIntent1"
onClickを指定することでボタンが押された時の処理を書くことができるようになります。
onClickに記述した名前がjavaファイルにメソッド名として記述されます。

MainActivity.java

                  
                    package com.example.testintent;

                    import androidx.appcompat.app.AppCompatActivity;

                    import android.content.Intent;
                    import android.os.Bundle;
                    import android.view.View;

                    public class MainActivity extends AppCompatActivity {

                        @Override
                        protected void onCreate(Bundle savedInstanceState) {
                            super.onCreate(savedInstanceState);
                            setContentView(R.layout.activity_main);
                        }

                        // 画面１から画面２へと画面遷移させるメソッド
                        public void onButtonIntent1(View v) {
                            Intent intent = new Intent(this, MainActivity2.class);
                            startActivity(intent);
                        }
                    }
onButtonIntent1メソッドを定義する

activity_main.xmlのButtonにつけたonClickの名前(onButtonIntent1)のメソッドを定義する。
Intent intent = new Intent(this, MainActivity2.class);で画面遷移に必要なIntentをインスタンス化し、
同時にどのアクティビティからどのアクティビティに遷移するのかを指定しています。
startActivity(intent);で先程インスタンス化したintentを実行するよう命令しています。
