Node.jsフレームワークExpress

Node.jsのWebアプリケーション作成用フレームワークであるExpressの導入方法から
Webアプリケーションの作り方までを記録していきます。

Expressとは

Node.jsのためのWebアプリケーションを作るためのフレームワーク
Expressは、Webアプリケーションとモバイルアプリケーション向けの
一連の堅固な機能を提供する最小限で柔軟なNode.js Webアプリケーション・フレームワークです。
Express公式サイトより https://expressjs.com/ja/

ExpressはNode.jsを使ってWebアプリケーションを作るためにとても便利なフレームワークです。 UIはpugファイルで定義できますし、作成段階にはサーバーを用意していなくてもlocalhost（仮想サーバー）を使って
Webアプリケーションをテストしながら作っていけることが個人的にとても気に入っています。

フレームワークとは

フレームワークとは、アプリケーション等のシステム開発時によく使う機能やディレクトリの構造等を用意してある骨組みです。
フレームワークを使うことで短時間での開発が可能になり、バグの低減にも繋がります。

フレームワークには固有のルールがあるのでプログラミング言語のスキルに加えフレームワークの知識も必要になりますが、
学習コストに見合う恩恵が得られると思います。

なぜExpressを選んだのか

Expressの特徴として、「高速」で「オープンソース」で「軽量」であることがあげられます。
その他にもExpressに関しての情報量が多いので独学でも勉強しやすい部類に入ると思われます。

Expressの導入からHello World!へ

Expressのインストール

Expressをインストールするには「npm」を使います。npmとは、「Node Package Manager」のことです。
当サイトの JavaScriptのページのデータベースの部分でも少し触れています。
ExpressはNode.jsのフレームワークなので、Node.jsとnpmがインストールされていることを前提とします。 

          
            // Expressをインストールするディレクトリを作ります。
            mkdir myApp

            // ディレクトリ myAppに移動します。
            cd myApp

            // npmを起動します。
            npm init

            // Expressをインストールします。
            npm install expressApp

            // ディレクトリが増えていることが確認できます。
            ls

            package.json package-lock.json app.js node.modules
app.jsのコード

ディレクトリの中のapp.jsを開くと以下のコードが記述されています。 

          
            const express = require('express');
            const app = express();

            app.get('/', (req, res) => {
              res.send('Hello World!');
            });

            app.listen(3000, () => {
              console.log('Example app listening on port 3000!');
            });
Expressの実行

ターミナルで下記のコマンドを打つことでExpressを実行することが可能です。 

          
            node app.js

Webブラウザを開き、localhost:3000と入力すると画面上に「Hello World!」が表示されます。
