Node.js入門！！

サーバーサイドで動くJavaScript ー Node.js ー
JavaScriptはブラウザでページに動きを追加するだけじゃない！！

Node.jsを使うことでより幅が広がるJavaScriptを楽しもう！！

Node.jsとは

「Node.js」はJavaScript実行環境

Node.jsはフレームワークやライブラリではなくJavaScriptの実行環境です。
「実行環境」という言葉を聞いて「？？？」という方のために。
僕もあやふやなところがあるのでそこを理解していきましょう。

実行環境とは

プログラムの実行するための環境です。
・・・そのままですね。
プログラムを動かすために必要なものが揃ってるパッケージとでもいえば分かりやすいでしょうか。
Node.js以外にも、Javaでいうところの「JRE」などプログラミング言語によってそれぞれ実行環境があります。
似たような言葉で「開発環境」とか「動作環境」というものがあるので混同しないよう注意しましょう。

「開発環境」とは、「動作環境」とは
  • 開発環境…動作環境に加えて開発するために必要なものが揃っているものです。
    例：JavaでいうところのJDK
  • 動作環境…言葉は似ていますが、全く意味が違ってきます。
    ソフトウェア等が正常に動作するために要求されるシステム条件＝必要スペック
    例：CPU：Intel Core i5以上、メモリ8GB以上、ストレージ1GB以上・・・
「フレームワーク」とは、「ライブラリ」とは
  • フレームワーク…目的に沿った全体の処理の流れが既に作ってあり、
    その中の一部を自分で実装して作り込んでいくもの
  • ライブラリ…プログラムを組む上で使いたい部品や便利なものが入っている道具箱のようなもの

Node.jsでできること

  • サーバー構築
    • データベースアクセス
  • gulp
  • Webサービスの開発
  • スマホアプリの開発
  • IoT関連の開発

node.jsの導入方法

上記環境で僕が実際にnode.jsを導入した方法を記載します。
後々のnode.jsのバージョン管理のことを考え、n packageでの導入を行いました。

  1. node.jsとnpmのインストール

    2. まずは下記の3つのコマンドを入力してnode.jsとnpm(Node Package Manager)をインストールしましょう。

              
            sudo apt-get update
            sudo apt-get install nodejs
            sudo apt-get install npm
  2. n packageを導入
    3.           
            sudo npm cache clean
            sudo npm install n -g

            // インストール後
            sudo n stable
            sudo ln /usr/local/bin/node /usr/bin/node
  3. 最初にインストールしたnode.js npmは使わないので、アンインストールしておく

    4. 最初にインストールされたnode.jsはバージョン指定したものではなく、その時に選ばれた（おそらく最新のもの）がインストールされているためアンインストールします。
    npmについても、後でnode.jsをインストールする際に同時にインストールされるようです。 

              
            sudo apt-get purge -y nodejs npm
  4. n packageでバージョンを管理する
    5.           
            sudo n インストールしたいバージョン
            sudo ln -sf /usr/local/bin/node /usr/bin/node
  5. node.jsが正常にインストールされたかを確認
    6.           
            node --version

            v12.18.3

Gulpを使ってPugやSassをコンパイルする

Gulpとは

Gulpとは、Node.jsベースのタスクランナー（自動化プログラムのひとつ）です。

Gulpでできること

  • HTML、CSS、JSファイルの圧縮
  • SassやLess、pugファイルのコンパイル
  • JavaScriptの構文チェック
  • …and more

Gulpの導入方法

GulpはNode.jsベースなので、「Node.js」と「npm」がインストールされていることが前提です。

Gulpをグローバル環境にインストール

初めてGulpを使う方はこの手順から行います。

  • Gulpをグローバル環境にインストール
    •           
            sudo npm install -g gulp
  • インストールされたGulpのバージョン確認
    •           
            // どちらのコマンドでも確認できます。
            gulp -v

            gulp --version
プロジェクトごとに必要な手順
  • プロジェクトのディレクトリ作成
    •           
            // プロジェクトディレクトリをつくる ディレクトリ名：testGulp
            mkdir testGulp
            // testGulpに移動
            cd testGulp
  • package.jsonファイルの作成
    •           
            npm init
  • Gulp本体をローカル環境にもインストール
    •           
            npm install --save-dev gulp
  • 必要なプラグインのインストール
    •           
            npm install --save-dev gulp-notify
            npm install --save-dev gulp-plumber
            npm install --save-dev gulp-pug
            npm install --save-dev gulp-sass
            npm install --save-dev gulp-autoprefixer
            npm install --save-dev gulp-uglify
            npm install --save-dev browser-sync
  • ディレクトリ構成
    • dist/
      • css/
      • js/
    • src/
      • js/
      • pug/
      • scss/
    • gulpfile.js
    • package.json
  • gulpfile.js
    •           
            const gulp          = require('gulp');
            const notify        = require('gulp-notify');
            const plumber       = require('gulp-plumber');
            const sass          = require('gulp-sass');
            const pug           = require('gulp-pug');
            const autoprefixer  = require('gulp-autoprefixer');
            const uglify        = require('gulp-uglify');
            const browserSync   = require('browser-sync');

            // setting : PATH
            const paths = {
              'root'      : './dist',
              'pug'       : './src/pug/**/*.pug',
              'html'      : './dist/**/*.html',
              'cssSrc'    : './src/scss/**/*.scss',
              'cssDist'   : './dist/css/',
              'jsSrc'     : './src/js/**/*.js',
              'jsDist'    : './dist/js/'
            }

            // gulpコマンドの省略
            const { watch, series, task, src, dest, parallel} = require('gulp');

            // Sass
            task('sass', function() {
              return(
                src(paths.cssSrc)
                .pipe(plumber({errorHandler: notify.onError("Error: <%= error.message %>")}))
                .pipe(sass({
                  outputStyle: 'expanded' // Minifyするなら 'compressed'
                }))
                .pipe(autoprefixer({
                  browsers: ['ie >= 11'],
                  cascade: false,
                  grid: true
                }))
                .pipe(dest(paths.cssDist))
              );
            });

            // Pag
            task('Pug', function () {
              return(
                src(paths.pug, '!./src/pug/**/*.pug')
                .pipe(plumber({errorHandler: notify.onError("Error: <%= error.message %>")}))
                .pipe(pug({
                  pretty: true,
                  basedir: "./src/pug"
                }))
                .pipe(dest(paths.html))
              );
            });

            // JS Compress
            task('js', function(){
              return(
                src(paths.jsSrc)
                .pipe(plumber())
                .pipe(uglify())
                .pipe(dest(paths.jsDist))
              );
            });

            // browser-sync
            task('browser-sync', () => {
              return browserSync.init({
                server:{
                  baseDir: paths.root
                },
                port: 8000,
                reloadOnRestart: true
              });
            });

            // browser-sync reload
            task('reload', (done) => {
              browserSync.reload();
              done();
            });

            // watch
            task('watch', (done) => {
              watch([paths.cssSrc], series('sass', 'reload'));
              watch([paths.jsSrc], series('js', 'reload'));
              watch([paths.pug], series('pug', 'reload'));
              done();
            });
            task('default', parallel('watch', 'browser-sync'));

node.jsでのデータベースアクセス

node.jsやSQLite3のデータベースファイルを格納しておくディレクトリ（フォルダ）をつくります。
ディレクトリの場所はお好みの場所で構いません。
不安な方は、HOMEディレクトリ直下に作ることをお勧めします。

ディレクトリを作ってそこに入ったら、ターミナルから次のコマンドを入力しnode.jsでSQLite3にアクセスできるようにします。 

        
          npm install sqlite3
SQLite3のデータベースファイル作成

SQLite3のコマンドでデータベースを作成しておきます。
データベースを作ったら、ctrl+dでsqlite3のコンソールから抜けましょう。 

        
          sqlite3 test.db
node.jsのファイル作成

データベースファイルを定義したフォルダの中にJavaScriptのファイルを作ります。 

        
          touch testAccess.js
データベースアクセスのコードを書こう！！
          
            // SQLite3を使う準備
            var sqlite3 = require('sqlite3').verbose();
            // DBファイルを指定
            var db = new sqlite3.Database('test.db');

            // DB操作
            db.serialize(() => {
              // テーブルを初期化
              dataInitialize();

              // データを格納
              insertData('Sasaki');
              insertData('Satou');
              insertData('Takahashi');

              // 格納したデータを抽出
              selectAll();

              // データを更新
              updateData('Takahashi', 'Tanaka');

              // 格納したデータを抽出
              selectAll();

              // データを削除
              deleteData('Tanaka');

              // 格納したデータを抽出
              selectAll();
            });
            db.close();


            /**
            * テーブルの初期化メソッド
            */
            function dataInitialize() {
              // テーブルを削除
              db.run('DROP TABLE testTable');

              // テーブルを定義
              db.run("CREATE TABLE testTable (id INTEGER PRIMARY KEY AUTOINCREMENT, name TEXT)");
              console.log("テーブルの初期化完了");
            }

            /**
            * SELECT文：DBのデータを抽出メソッド
            */
            function selectAll() {
              db.each("SELECT * FROM testTable", (error, row) => {
                if(error) {
                  console.error("Error", error);
                  return;
                }
                console.log(row.id + ' | ' + row.name);
              });
            }

            /**
            * INSERT文：データを新規に格納するメソッド
            * @param {新規追加の名前} name 
            */
            function insertData(name) {
              const stmt_insert = db.prepare("INSERT INTO testTable (name) VALUES (?)");
              stmt_insert.run(name);
              stmt_insert.finalize();
              console.log("データの格納完了");
            }

            /**
            * 既存のデータを更新するメソッド
            * @param {変更前の名前} oldName 
            * @param {変更後の名前} newName 
            */
            function updateData(oldName, newName) {
              const stmt_update = db.prepare("UPDATE testTable SET name = ? WHERE name = ?");
              stmt_update.run([newName, oldName]);
              stmt_update.finalize();
              console.log("データの更新完了");
            }

            /**
            * データを削除するメソッド
            * @param {削除する名前} name 
            */
            function deleteData(name) {
              const stmt_delete = db.prepare("DELETE FROM testTable WHERE name = ?");
              stmt_delete.run(name);
              stmt_delete.finalize();
              console.log("データの削除完了");
            }
データベースの種類とパスの指定

var sqlite3 = require('sqlite3').verbose();
「SQLite3データベースを使います」という宣言を変数sqlite3に格納しています。
var db = new sqlite3.Database('test.db');
「test.db」というSQLite3データベースを定義して変数dbに格納しています。
この2行を記述することで、データベースの種類とパス（データベースの場所と名前）を指定します。

データベースアクセスの処理を記述

db.serialize(() => {...});
db.close();
このブロックでデータベースへのデータの追加・更新・削除・読込等の処理をどういった順序で行うのかを記述しています。
ここで呼び出している関数はブロックの後ろの方で記述しています。

テーブルの初期化メソッド

今回のコードはテスト用コードということもあり、DROP TABLE文によるデーブルの削除と
CREATE TABLE文によるテーブルの定義を組み合わせた形でテーブルの初期化メソッドを定義しました。

SELECT文：データの読込メソッド

SELECT文の場合のみコードの形が少し違っているので、注意が必要です。 for文等の繰り返し文をつかなくてもdb.each("SELECT文", (error, row) => {});の記述だけで
SELECT文に応じたデータを抽出してくれます。

INSERT文・UPDATE文・DELETE文のメソッド

INSERT文・UPDATE文・DELETE文の3つは共通点のある記述方法です。 db.prepare();というメソッドを使うことでプレースホルダ(?)を用いてSQL文を記述することができます。
そのプレースホルダに引数をわたしてメソッドを定義しています。
それぞれのメソッドによってプレースホルダの数や記述箇所が異なるので注意が必要です。
