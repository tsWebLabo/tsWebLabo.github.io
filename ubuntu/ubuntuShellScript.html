ShellScriptはLinuxコマンドより少し敷居が高いかもしれません。
ルーティンとしてLinuxコマンドを走らせたい時などにはShellScriptが便利です。
僕もまだ未熟な分野ですが、一緒に勉強していきましょう！！

ShellScript

ShellScriptは複数のLinuxコマンドをひとつのファイルに記述して実行できるようにしたもの。
ターミナルでUbuntuをアップデートするなど、複数のコマンドをルーティンで実行させたい時に有効。

ShellScriptのファイルの作成から実行まで

  • 「****(ファイル名).sh」の形でファイルを作成
    •             
              touch ****.sh
  • ファイルにShellScriptを記述
    •             
              #!/bin/bash
              sudo apt update;

    必ず1行目に#!/bin/bashというシバンと呼ばれる定型文を書く

  • ファイルに実行権限を付与
    •             
              chmod +x ****.sh
  • ShellScriptを実行
    •           
            sh ****.sh

ShellScriptの例：Ubuntuアップデート

Ubuntuのアップデートをターミナルで行う時、Linuxコマンドで行うと６つのコマンドを「&&」で繋げて実行するか
別々に実行させる必要があり、かつ定期的に同じことをする必要が出てくるためShellScriptにしておくと効率的にアップデートを行えます。
このファイルを実行させると、ファイル内にsudo（管理者権限コマンド）があるため、パスワードを尋ねられます。 

        
          #!/bin/bash

          # Ubuntu本体とインストールされているソフトウェアのメンテナンス・アップデート

          # 最新のアップデートを確認
          sudo apt-get update;

          # 確認したアップデートを適用
          sudo apt-get -y upgrade;

          # ディストリビューションを最新のものに適用
          sudo apt-get -y dist-upgrade;

          # aptのパッケージキャッシュクリア
          sudo apt-get clean;

          # 依存関係解決後に使わなくなったパッケージを削除
          sudo apt-get -y autoremove;

          # 他のパッケージの依存関係を満たすため自動的にインストールされ、不要になったパッケージを削除
          sudo apt-get -y autoclean;
