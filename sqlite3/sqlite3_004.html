SQLite3

SQLite3は、サーバーを用意する必要なく気軽に使えるデータベースです。
SQLite3を通してデータベースとSQL文に触れてみませんか？

テーブルにデータを追加するためのSQLがINSERT文です。
INSERT文を効果的に使ってテーブルにデータを追加していきましょう。

データの追加（INSERT文）

テーブルにデータを追加するにはINSERT文を使います。
全てのカラムにデータを追加する方法と、カラムを指定してデータを追加する方法があります。

全てのカラムに値を指定してデータを追加する方法

書式

        
          INSERT INTO テーブル名 VALUES(値1, 値2, ....);

例文
テスト用にテーブルを作成します。 

        
          CREATE TABLE testUser(id INTEGER, name TEXT, age INTEGER, address TEXT);

このテーブルには４つのカラムがあります。
このテーブルにデータを追加する場合、4つの値をカラムの順番に沿って追加します。
例えば、id=1, name='Yamada', age=19, address='Tokyo'だとします。 

        
          INSERT INTO testUser VALUES (1, 'Yamada', 19, 'Tokyo');

引き続き以下のデータを追加してみます

        
          INSERT INTO testUser VALUES (2, 'Mori', 32, 'Kyoto');
          INSERT INTO testUser VALUES (3, 'Ueda', 27, 'Tokyo');
          INSERT INTO testUser VALUES (4, 'Mitsui', 20, 'Nagoya');

データが追加されたので、SELECT分を使ってデータを取得してみます。 

        
          SELECT * FROM testUser;
        
      
        
          id          name        old         address   
          ----------  ----------  ----------  ----------
          1           Yamada      19          Tokyo     
          2           Mori        32          Kyoto     
          3           Ueda        27          Tokyo     
          4           Mitsui      20          Nagoya

特定のカラムに値を指定してデータを追加する方法

書式

        
          INSERT INTO テーブル名(カラム1, カラム2, ....) VALUES(値1, 値2, ....);

例文
テスト用にテーブルを作成します。 

        
          CREATE TABLE testUser2(id INTEGER, name TEXT, age INTEGER, address TEXT DEFAULT 'NO VALUE');

テーブルには３つのカラムがあり、addressカラムにはDEFAULT制約が設定されています。 まずは３つのカラム全てに値を指定してデータを追加してみます。 

        
          INSERT INTO testUser2(id, name, address) VALUES(1, 'Honda', 'Tokyo');

次に２つのカラムだけに値を指定してデータを追加してみます。 

        
          INSERT INTO testUser2(id, name) VALUES(2, 'Okada');
          INSERT INTO testUser2(id, address) VALUES(3, 'Mori');

データが追加されたので、SELECT分を使ってデータを取得してみます。 

        
          SELECT * FROM testUser2;
        
      
        
          id          name        address   
          ----------  ----------  ----------
          1           Honda       Tokyo     
          2           Okada       NO VALUE  
          3                       Mori

２つ目のデータを追加した時には、addressカラムの値を指定していなかったのですが、
addressカラムにはDEFAULT制約が設定されているためデフォルト値である'no value'が格納されています。

３つ目のデータを追加した時には、nameカラムの値を指定していなかったので、デフォルト値であるNULLが格納されています。
（NULLは画面上では何も表示されません）

尚、データを追加する際に記述するカラムの順番はテーブルで定義されているカラムの順番通りである必要はありません。
次の２つのINSERT文は同じ結果となり、エラーにもなりません。 

        
          INSERT INTO user(id, name) VALUES(2, 'Ueda');
          INSERT INTO user(name, id) VALUES('Ueda', 2);

値を指定せずにデータを追加する方法

値を指定せずにデータを追加することもできます。
この場合、全てのカラムにデフォルト値が格納されます。

書式

        
          INSERT INTO テーブル名 DEFAULT VALUES;

先程のテーブル'testUser2'をそのまま利用します。

        
          SELECT * FROM testUser2;
        
      
        
          id          name        address   
          ----------  ----------  ----------
          1           Honda       Tokyo     
          2           Okada       NO VALUE  
          3                       Mori

値を指定せずにデータを追加します。

        
          INSERT INTO testUser2 DEFAULT VALUES;
        
      
        
          SELECT * FROM testUser2;
        
      
        
          id          name        address   
          ----------  ----------  ----------
          1           Honda       Tokyo     
          2           Okada       NO VALUE  
          3                       Mori      
                                  NO VALUE

追加されたデータは値が指定されていないので、
DEFAULT制約がされていないカラム（id, name）にはNULLが格納される。
DEFAULT制約が設定されているカラム（address）にはデフォルト値が格納されました。
