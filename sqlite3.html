SQLite3

SQLite3を通して気軽にデータベースとSQL文に触れてみよう！！

It uses utility classes for typography and spacing to space content out within the larger container.

SQLite3のインストールとコマンドラインツールの起動

・SQLite3のインストール

ターミナルで以下のコマンド入力することでSQLite3のインストールを行えます

          
            sudo apt install sqlite3

・コマンドラインツールの起動

コマンドラインツールを使うことで、SQLite3を直接、対話的に使うことができます。

          
            sqlite3 sample.db

実行結果

          
            SQLite version 3.22.0 2018-01-22 18:45:57
            Enter ".help" for usage hints.
            sqlite>

データベースの操作

データベース作成・接続

          
            sqlite3 sample.sqlite3

上記コマンドはSQLite3のコマンドライン起動コマンドと同じものです。
このコマンドを実行した際、指定したデータベースが存在すればそのデータベースに接続し、
指定したデータベースが存在しなかった場合はそのデータベース名で新規ファイルが作成されコマンドラインツールが開きます。

sqlite3での拡張子

データベース名については、拡張子を付けなくてもデータベースとして使うことは可能です。
ですが、ひとつのプロジェクトには様々なプログラムファイルが格納してあり、かつデータベースの種類も多数あることから、
それを明示的にするために下記のような拡張子をつけることが通例となっているようです。

  • sample.sqlite3
  • sample.sqlite
  • sample.db

データベースのバックアップ

SQLite3はひとつのデータベースを１ファイルとして管理しているので、
ファイル名を変えてコピーするだけでデータベースのバックアップをとることができます。

データベースの削除

バックアップの項目と同じ理由で、データベースのファイルを削除することでデータベースを削除することができます。

テーブルの操作

テーブル作成

SQLのCREATE文でテーブルを作成します。
新規データベースでこの操作を行った時点で、Nutilus(ファイルマネージャ)にデータベースのファイルが表示されます。 

          
            CREATE TABLE testTable001(id, name);

テーブルの確認

テーブルの確認方法は２種類あります。
状況に合わせて使い分けましょう。

  • .tables

    • データベースに格納されたテーブルを確認したい時に使う

                  
                .tables

    実行結果

                  
                testTable001  testTable002
  • .schema

    • データベースに格納されたテーブルの構造を確認することができる。

                  
                .schema

    実行結果

                  
                CREATE TABLE testTable001(id, name);
                CREATE TABLE testTable002(id INTEGER PRIMARY KEY, name TEXT NOT NULL);

テーブル名の変更

テーブル名を変更するには下記のSQL文を使います。

          
            ALTER TABLE 現在のテーブル名 RENAME TO 新しいテーブル名;

前述のテーブル"testTable002"を"myfriends"に変更してみます。

          
            ALTER TABLE testTable002 RENAME TO myfriends;

テーブル名が変更されているかを.tablesコマンドで確認します。

          
            .tables

            myfriends  testTable001

カラムの追加

既につくってあるテーブルに後からカラムを追加することもできます。
カラムを追加するには、下記のALTER TABLE文を使います。 

          
            ALTER TABLE テーブル名 ADD COLUMN カラム名 [データ型];

追加したカラムはテーブルの末尾に追加されます。
また、カラムを追加する際には次の条件を満たしていなければなりません。

  • PRIMARY KEY や UNIQUE制約は設定できない。
  • DEFAULT制約を設定する時は、CURRENT_TIME/CURRENT_DATE/CARRENT_TIMESTAMPは指定できない。
  • NOT NULL制約を設定する時は、NULL以外のデフォルト値の設定が必要。

現在のテーブルを確認してみます。

          
            .schema

            CREATE TABLE myfriend(id INTEGER PRIMARY KEY, name TEXT NOT NULL);
            CREATE TABLE testTable001(id, name);

テーブル”myfriends”に新しいカラム”address”を追加してみます。

          
            ALTER TABLE myfriends ADD COLUMN address TEXT;

テーブルを確認すると、新しいカラム”address”が追加されたことが分かります。

          
            .schema

            CREATE TABLE myfriends(id INTEGER PRYMARY KEY, name TEXT NOT NULL, address TEXT);
            CREATE TABLE testTable001(id, name);

カラムの削除

SQLite3では、カラムを削除するコマンドが無いようです。
なので、任意のカラムを削除したい時は次の手順を行う必要があります。

  1. 元テーブル（削除対象のカラムが入っているもの）を用意する。
  2. 元テーブルのコピーを削除対象のカラムを除いて作成する。（必然的にテーブル名が違うものになる）
  3. 元テーブルを削除する。
  4. コピーしたテーブル名を元テーブルの名前と同じものに変更する。

テーブルの削除

テーブルを削除するにはDROP TABLE文を使います。

.tablesコマンドで現在作成済みのテーブルを確認します。

          
            .tables

            myfriends  testTable001

テーブル”myfriends”を削除します。

          
            DROP TABLE myfriends;

テーブルを確認してみます。

          
            .tables

            testTable001

データの追加（INSERT文）

テーブルにデータを追加するにはINSERT文を使います。
全てのカラムにデータを追加する方法と、カラムを指定してデータを追加する方法があります。

全てのカラムに値を指定してデータを追加する方法

書式

          
            INSERT INTO テーブル名 VALUES(値1, 値2, ....);

例文
テスト用にテーブルを作成します。 

          
            CREATE TABLE testUser(id INTEGER, name TEXT, age INTEGER, address TEXT);

このテーブルには４つのカラムがあります。
このテーブルにデータを追加する場合、4つの値をカラムの順番に沿って追加します。
例えば、id=1, name='Yamada', age=19, address='Tokyo'だとします。 

          
            INSERT INTO testUser VALUES (1, 'Yamada', 19, 'Tokyo');

引き続き以下のデータを追加してみます

          
            INSERT INTO testUser VALUES (2, 'Mori', 32, 'Kyoto');
            INSERT INTO testUser VALUES (3, 'Ueda', 27, 'Tokyo');
            INSERT INTO testUser VALUES (4, 'Mitsui', 20, 'Nagoya');

データが追加されたので、SELECT分を使ってデータを取得してみます。 

          
            SELECT * FROM testUser;
          
        
          
            id          name        old         address   
            ----------  ----------  ----------  ----------
            1           Yamada      19          Tokyo     
            2           Mori        32          Kyoto     
            3           Ueda        27          Tokyo     
            4           Mitsui      20          Nagoya

特定のカラムに値を指定してデータを追加する方法

書式

          
            INSERT INTO テーブル名(カラム1, カラム2, ....) VALUES(値1, 値2, ....);

例文
テスト用にテーブルを作成します。 

          
            CREATE TABLE testUser2(id INTEGER, name TEXT, age INTEGER, address TEXT DEFAULT 'NO VALUE');

テーブルには３つのカラムがあり、addressカラムにはDEFAULT制約が設定されています。 まずは３つのカラム全てに値を指定してデータを追加してみます。 

          
            INSERT INTO testUser2(id, name, address) VALUES(1, 'Honda', 'Tokyo');

次に２つのカラムだけに値を指定してデータを追加してみます。 

          
            INSERT INTO testUser2(id, name) VALUES(2, 'Okada');
            INSERT INTO testUser2(id, address) VALUES(3, 'Mori');

データが追加されたので、SELECT分を使ってデータを取得してみます。 

          
            SELECT * FROM testUser2;
          
        
          
            id          name        address   
            ----------  ----------  ----------
            1           Honda       Tokyo     
            2           Okada       NO VALUE  
            3                       Mori

２つ目のデータを追加した時には、addressカラムの値を指定していなかったのですが、
addressカラムにはDEFAULT制約が設定されているためデフォルト値である'no value'が格納されています。

３つ目のデータを追加した時には、nameカラムの値を指定していなかったので、デフォルト値であるNULLが格納されています。
（NULLは画面上では何も表示されません）

尚、データを追加する際に記述するカラムの順番はテーブルで定義されているカラムの順番通りである必要はありません。
次の２つのINSERT文は同じ結果となり、エラーにもなりません。 

          
            INSERT INTO user(id, name) VALUES(2, 'Ueda');
            INSERT INTO user(name, id) VALUES('Ueda', 2);

値を指定せずにデータを追加する方法

値を指定せずにデータを追加することもできます。
この場合、全てのカラムにデフォルト値が格納されます。

書式

          
            INSERT INTO テーブル名 DEFAULT VALUES;

先程のテーブル'testUser2'をそのまま利用します。

          
            SELECT * FROM testUser2;
          
        
          
            id          name        address   
            ----------  ----------  ----------
            1           Honda       Tokyo     
            2           Okada       NO VALUE  
            3                       Mori

値を指定せずにデータを追加します。

          
            INSERT INTO testUser2 DEFAULT VALUES;
          
        
          
            SELECT * FROM testUser2;
          
        
          
            id          name        address   
            ----------  ----------  ----------
            1           Honda       Tokyo     
            2           Okada       NO VALUE  
            3                       Mori      
                                    NO VALUE

追加されたデータは値が指定されていないので、
DEFAULT制約がされていないカラム（id, name）にはNULLが格納される。
DEFAULT制約が設定されているカラム（address）にはデフォルト値が格納されました。

データの更新（UPDATE文）

テーブルに格納されているデータを新しい値に更新するにはUPDATE文を使います。

書式

          
            UPDATE テーブル名 SET カラム名1 = 値1, カラム2 = 値2, ... WHERE 条件式;

まず更新の対象となるデータをWHERE句の条件式を使って指定します。
条件式に一致するデータが複数の場合、複数のデータがそれぞれ更新されます。
※WHERE句を省略した場合、テーブルに含まれる全てのデータが更新されます

次のテーブルを例にUPDATE文がどのようなものか見ていきます。 

          
            CREATE TABLE staff (id INTEGER, name TEXT, unit TEXT, flag TEXT);
            INSERT INTO staff VALUES (1, 'Yamada', 'Sales', 'Yes');
            INSERT INTO staff VALUES (2, 'Honda', 'Office', 'No');
            INSERT INTO staff VALUES (3, 'Nakai', 'Office', 'Yes');
            INSERT INTO staff VALUES (4, 'Suzuki', 'Tech', 'No');
            INSERT INTO staff VALUES (5, 'Itou', 'Sales', 'No');
          
        
          
            SELECT * FROM staff;

            id          name        unit        flag      
            ----------  ----------  ----------  ----------
            1           Yamada      Sales       Yes       
            2           Honda       Office      No        
            3           Nakai       Office      Yes       
            4           Suzuki      Tech        Yes       
            5           Itou        Sales       No

ひとつのデータを更新

idカラムの値が3のデータのnameカラムの値を変更

          
            UPDATE staff SET name = 'Nakajima' WHERE id = 3;
          
        
          
            SELECT * FROM staff;

            id          name        unit        flag      
            ----------  ----------  ----------  ----------
            1           Yamada      Sales       Yes       
            2           Honda       Office      No        
            3           Nakajima    Office      Yes       
            4           Suzuki      Tech        Yes       
            5           Itou        Sales       No

複数のデータを一度に更新する場合

unitカラムの'Offce'という値を'Desk'に変更します。

          
            UPDATE staff SET unit = 'Desk' WHERE unit = 'Office';
          
        
          
            SELECT * FROM staff;
            
            id          name        unit        flag      
            ----------  ----------  ----------  ----------
            1           Yamada      Sales       Yes       
            2           Honda       Desk        No        
            3           Nakajima    Desk        Yes       
            4           Suzuki      Tech        Yes       
            5           Itou        Sales       No

全てのデータを一度に更新する場合

flagカラムの値を'unknown'に変更してみます。

          
            UPDATE staff SET flag = 'unknown';
          
        
          
            SELECT * FROM staff;

            id          name        unit        flag      
            ----------  ----------  ----------  ----------
            1           Yamada      Sales       unKnown   
            2           Honda       Desk        unKnown   
            3           Nakajima    Desk        unKnown   
            4           Suzuki      Tech        unKnown   
            5           Itou        Sales       unKnown

データの削除（DELETE文）

テーブルからデータを削除するには、DELETE文を使います。

書式

          
            DELETE FROM テーブル名 WHERE 条件式;

まず削除の対象となるデータを条件式を使って指定します。
条件式に一致するデータが複数の場合は複数のデータがまとめて削除されることになります。
WHERE句を省略してしまった場合はテーブルに含まれる全てのデータが削除されます。

下記のテーブルを例にDELETE文を試してみます。 

          
            CREATE TABLE testUser3 (id INTEGER, name TEXT, age INTEGER);
            INSERT INTO testUser3 VALUES (1, 'Yamada', 32);
            INSERT INTO testUser3 VALUES (2, 'Suzuki', 15);
            INSERT INTO testUser3 VALUES (3, 'Honda', 18);
            INSERT INTO testUser3 VALUES (4, 'Endou', 24);
            INSERT INTO testUser3 VALUES (5, 'Furuta', 19);
          
        
          
            SELECT * FROM testUser3;

            id          name        age       
            ----------  ----------  ----------
            1           Yamada      32        
            2           Suzuki      15        
            3           Honda       18        
            4           Endou       24        
            5           Furuta      19

ageカラム20より小さいデータを削除します。

          
            DELETE FROM testUser3 WHERE age < 20

実行結果

          
            SELECT * FROM testUser3;

            id          name        age       
            ----------  ----------  ----------
            1           Yamada      32        
            4           Endou       24

WHERE句を省略して全てのデータを削除してみます。

          
            DELETE FROM testUser3;

実行結果

          
            SELECT * FROM testUser3;

データの取得（SELECT文）

テーブルの中のどのカラムの値を取得するのかをSELECTの後に記述し、
取得するテーブル名をFROMの後に記述します。

書式

          
            SELECT カラム名 FROM テーブル名;

テスト用に下記のテーブルを作成しておきます。 

          
            CREATE TABLE testUser4 (id INTEGER, name TEXT, address TEXT);
            INSERT INTO testUser4 VALUES (1, 'Yamada', 'Tokyo');
            INSERT INTO testUser4 VALUES (2, 'Kumada', 'Osaka');
            INSERT INTO testUser4 VALUES (3, 'Hoshi', 'Kyoto');
            INSERT INTO testUser4 VALUES (4, 'Endou', 'Tokyo');
            INSERT INTO testUser4 VALUES (5, 'Suzuki', 'Osaka');

任意のカラムのデータを取得

任意に複数のカラムの値を取得する時はカンマ(,)で区切ってカラムを指定します。

書式

          
            SELECT カラム1, カラム2, .... FROM テーブル名;
          
        
          
            SELECT id, name FROM testUser4

            id          name      
            ----------  ----------
            1           Yamada    
            2           Kumada    
            3           Hoshi     
            4           Endou     
            5           Suzuki

全てのカラムのデータを取得

テーブルに含まれるすべてのカラムの値を取得したい場合は、
カラム名の代わりにアスタリスク（*）を記述すると全てのカラムを指定したことと同じ結果になります。

書式

          
            SELECT * FROM テーブル名;
          
        
          
            SELECT * FROM testUser4;

            id          name        address   
            ----------  ----------  ----------
            1           Yamada      Tokyo     
            2           Kumada      Osaka     
            3           Hoshi       Kyoto     
            4           Endou       Tokyo     
            5           Suzuki      Osaka
